Preguntas sobre fragmentos comunes:

- ¿Hay un catálogo de fragmentos? Si lo hay, ¿quién puede consultar ese
  catálogo? ¿Cuántos fragmentos hay? ¿Hay un buscador?

- ¿Hay un versionado de los fragmentos?

- ¿Puedo mezclar campos propios con fragmentos comunes?

- ¿Están agrupados o clasificados los fragmentos de alguna forma? Por
  área temática o cualquier otra clasificación?

- ¿Un fragmento común puede contener otros fragmentos comunes?

- ¿Hay reactividad? Por ejemplo, al cambiar el combo de la lista cambiar
  el combo de municipios?

- ¿Hay posibilidad de subdividir el fragmento y/o formulario?  Por
  ejemplo, un botón para seleccionar si una comunicación debe realizarse
  vía postal o por correo electrónico, y en ese caso preguntar solo los
  datos correspondientes a cada caso.

- ¿Funcionan los fragmentos si no está habilitado Javascript?

- ¿Hay formularios de múltiples pasos? Si es así, ¿dónde se guardan los
  datos del formulario de forma temporal?
  
- ¿Se puede saber quien está usando qué fragmento?

- ¿Dónde encaja el concepto de Formulario? ¿Es un conjunto de fragmentos
  comunes? ¿Cuantos formularios hay?

- ¿Puedo usar un *webform* para cualquier cosa? Específicamente, puedo usar
  un *webform* para algo que no sea un procedimiento administrativo? Por
  ejemplo, un formulario de búsqueda? ¿Puedo usar los *webforms* en
  aplicaciones propias?

- ¿Cuánto tiempo lleva aproximadamente diseñar un formulario tipo, o un
  fragmento común?

- ¿Quién diseña los formularios? Como es el proceso de edición,
  aprobación, cambios, etc...

- ¿Qué lenguaje/sistema/mecanismo se usa para la validación de las
  entradas? Por ejemplo, un campo que solo pueda valer entre 7 y 21.
  ¿Hay algo para las validaciones que afecten a varios campos? ¿Se valida
  en *fronternd*, *backend* o ambos?
